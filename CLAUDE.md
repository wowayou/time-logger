# CLAUDE.md — 时间尺 AI 约定 + 维护规范

## 项目性质

时间尺是**单文件零依赖 PWA**（`index.html` + `sw.js` + `manifest.webmanifest` + `icon.svg`）。它是求职期间的测量仪器，不是产品。

**铁律：零依赖 / 单文件 / 无构建。** 不引入 npm、不加构建步骤、不拆分文件。

---

## 铁律 — SW 缓存号必须 bump

改动 `index.html`、`sw.js`、`manifest.webmanifest` 后，**必须**把 `sw.js` 第 1 行的 `CACHE` 从 `timelog-vN` 升到 `timelog-v(N+1)`。

不升号 → 已装用户永远看不到更新（SW 继续提供旧缓存）。

若新增运行时文件（如新图片），同步加进 `sw.js` 第 2 行的 `FILES` 数组。文档 `.md` 不进缓存。

## 代码约定

- **颜色只用 CSS 变量**（`--bg`、`--accent` 等），禁止散落硬编码色值（按钮上的 `#fff` 文字除外）
- **日期值统一 `YYYY-MM-DDTHH:mm`**
- **尺子未记录段用 `--track`**，不用 `--border`（亮色下 border 太浅，track 专门调过对比度）
- 纯原生 JS，无框架，无构建，不加 `type="module"`
- 公开发布只发布 `time-logger/` 独立仓库；不要提交导出的 JSON、真实记录截图、父目录 `toolkit/` 或 `archive/`

最大风险 = 用打磨工具逃避面试推进。如果用户或 AI 在没有充分求职进展的情况下要求继续打磨功能，请明确指出这一风险。

## 改动自测清单

每次改完在本地跑一遍：

1. `python3 -m http.server` 打开 `index.html`
2. **亮色**：系统切亮/暗自动跟随；自动/亮色/暗色手动覆盖且刷新后保持；尺子"未记录"段亮暗都清晰
3. **滚轮**：触控/鼠标拖拽有吸附；方向键移一格；"现在"复位；桌面精确输入框可用；新增和编辑都生效；保存后 ts 正确
4. **补录**：设数小时前的时间 → 自动排进正确位置；>3h 空档显示"未记录"
5. **导出/导入**：下载得到带时间戳 .json；改坏一条再导入 → 按 id 合并不丢新增；结构非法时报错不覆盖；"复制"仍可喂 AI
6. **PWA 更新链路**：改 `index.html` 后升 CACHE 号 → 刷新两次确认看到新版
7. 刷新后数据仍在（localStorage）
