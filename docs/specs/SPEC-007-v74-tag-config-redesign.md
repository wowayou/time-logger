status: ready（建议在 SPEC-004 冷白合并后开工，避免亮色令牌冲突；与 SPEC-006 无冲突可并行）
owner: 执行方认领后填分支名
验收人: Fable

## 背景

维护者 2026-07-24 真机反馈两点：① 标签高级设置视觉可以优化（现状：说明长段落 + 原生 `<select>` + 原生 checkbox，与应用自定义语言脱节）；② 主线当前完全不可编辑（sheet 里只有一行静态「主线历史」），要求分析是否放开。

**主线编辑的裁决（Fable 定案，即 D7「主线高级设置交由后续调研」的答案）**：

| D7 列的四类动作 | 裁决 | 理由 |
|---|---|---|
| 改名 | **做** | chip 改名的历史迁移机制已存在（`saveTagConfig`），扩展到 mainline 数组是同一机制；不做反而是不对称的怪状 |
| 改桶 | **不做** | 主线＝job 桶的定义本身；「主线改桶」实为「降级出主线」，语义另议，v1 不碰 |
| 停用/删除 | **不做** | 历史记录引用主线历史名，删除会产生孤儿标签 |
| `longOk` | **做** | 主线工作段 >3h 是常态，目前每段都要确认是真实摩擦；per-tag 豁免机制现成 |

另加一个动作：**「设为当前主线」**——`config.mainline` 本就是 unshift 语义的历史数组（`storage.js:138` `addMainlineTag`），用户今天只能靠「录入时输入新 job 标签」隐式切换（维护者的 `#app推进` 就是这么来的）；给历史里的名字一个显式「设为当前」按钮只是把既有语义放上台面。

## 设计方向（Fable 定案，执行按此实施；全部在既有令牌系统内，不引入新视觉身份）

**签名手法：把时间轴的桶色竖脊带进设置行。** 每个标签行左缘 4px 通高竖脊，颜色＝所属桶（与日视图 `data-b` 同源令牌）——结构即信息：不用读控件就知道这行属于哪个桶；主线行用 job 紫脊，是整张 sheet 里唯一的紫，天然成为视觉锚点。

1. **分组**：主线 / 维持 / 偏航 三个 `cell-group`（块级流，P21/P34 红线适用），组标题用现有分节样式。
2. **主线组**：当前主线行置顶（紫脊 + 名称输入框 + `longOk` 开关）；主线历史各名一行（紫脊淡化 55% 透明——与 hero 比例条维持段同一手法），行尾「设为当前」文字按钮（`mini-btn` 语法，44px 热区）；无删除。
3. **维持/偏航行**：名称输入 + **两段式分段控件（维持｜偏航）替换原生 `<select>`**——只有两个选项，segmented control 比弹出式 select 更直接，且消灭 iOS 原生弹层的语言断裂；选中态用桶色 tint（`--maintain-bg`/`--leak-bg`）。切桶时行的竖脊颜色即时跟随（即时反馈，结构与控件说同一件事）。
4. **`超长段免确认`**：原生 checkbox 换应用样式开关行（44px 命中、文案不变）；保持每行第二行的位置。
5. **说明文案**：现有四句长段落压缩为组内一行帮助小字各司其职（「改名会同步迁移历史记录」放主线/chip 名称输入附近；「录入时先选桶…」移交帮助页）；文案句式主动、说人话。
6. **不动**：sheet 头部 C 语法（取消/保存）、returnToMore 返回栈、`saveTagConfig` 的校验语义（重名/主线-chip 冲突检查扩展到主线改名方向即可）、16px 输入字号、focus trap。

## 功能语义（必须精确实现）

- 主线改名：迁移所有历史记录的该标签名（与 chip 改名同一事务语义）；改名后 `mainline` 数组内该名同步替换；与 chips/其它 mainline 名冲突时 inline 报错阻止保存（沿用 `showInlineError`）。
- 设为当前：把该名移到 `mainline[0]`（unshift 语义），不产生记录变化；FAB 续记与新建表单的默认主线随之更新。
- 主线 `longOk`：主线名进入 per-tag 豁免集合的存取路径（现 config 只对 chips 存 longOk——需要为 mainline 名扩展存储位；**导入合并兼容**：老备份无此字段照常读入，`validateImportData` 相应放宽/校验新字段）。
- 录入时隐式新增主线（`addMainlineTag`）行为完全不变。
