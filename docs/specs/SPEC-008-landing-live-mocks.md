# SPEC-008 · 着陆页活体特性演示（纯 HTML/CSS 组件替代/补充截图）

status: ready（依赖 SPEC-005 已合并的 landing v2 作底；零运行时改动，无版本号）
owner: 执行方认领后填分支名
验收人: Fable

## 背景

维护者 2026-07-24 提出：着陆页特性区目前只有两张白名单 demo PNG 可配图，另两条特性只有文字；参考 https://restcal.abohack.com/ 的做法——能否不靠截图动态展示。Fable 对 restcal 的技术分析：**它的 hero 是纯 HTML/CSS 手搭的假应用窗口**（`hero-window`/`hero-window-bar` 结构 + 20 个内联 SVG，全站仅 1 张 `<img>`）——不是截图，也不是 iframe，是用页面自己的样式系统搭出来的**示意组件**。

三个方案的诚实对比（分析结论，spec 采纳方案 B）：

| 方案 | 自动跟随应用更新？ | 代价 |
|---|---|---|
| A. 截图（现状） | 否，需重新截图 | PNG 白名单仪式；DPI/主题固定 |
| B. **CSS 示意组件** | 否，但**跟随着陆页令牌**（双主题自适应、任意 DPI 清晰、零 PNG） | 手工维护，是「示意」不是「实况」 |
| C. iframe 嵌真应用 demo 模式 | 是 | 需要运行时加 demo 数据模式（同 origin iframe 还会读到访客自己的 localStorage，隐私边界复杂）——**登记不做**，将来有真实需要另立规格 |

## 范围

在 SPEC-005 的四个特性节里，把配图从 PNG 换成/补上内联 CSS 示意组件（`site/index.html` 单文件内实现，零 JS 或最小 JS，双主题跟随 `prefers-color-scheme`）：

1. **「一天的形状，一眼看清。」**——迷你连续日志：6–8 行「时间｜内容｜时长」示意行，左缘 4px 桶色竖脊（紫/绿/棕/灰与应用令牌同值），竖读色序呈现一天形状；内容用固定虚构 demo 文案（与 docs/assets 截图的 demo 数据风格一致，不用真实记录）。
2. **「没记的时间，不会被抹掉。」**——迷你 hero 结论卡：主线大数字 + 比例条（四段色带含灰色未记录段）+ 一行「未记录 ~1h12min」高亮，示意「不凑 100%」。
3. **「你的数据只在你手里。」**——一张「备份 JSON」示意卡：等宽字体的 JSON 片段（字段名真实、值虚构），角标「导出 · 本地文件」。
4. **「偏航不是错误。」**——四枚桶色票据（主线/维持/偏航/未记录）横排，偏航条目下加一行小字「不做道德评判，只标注偏离」。
5. 两张真实 demo PNG **保留**但降级为「真实界面」小节或链接（执行方按版式取舍，PR 说明）；`build_site.py` 的 assets 复制逻辑相应保留或收敛。
6. **诚实标注**：示意组件区域加一行小字「界面示意，实际以应用为准」——它们是 stylized mock，不冒充截图。

## 设计纪律

- 全部颜色取自着陆页已有 CSS 变量（桶色直接用应用令牌同值：`#5745c7`/`#237a52`/`#9a4f1e`/灰 track），不引新色。
- 组件是**示意**不是像素复刻：结构对（竖脊/三列行/比例条），细节从简；不试图逐像素模仿应用，避免每次应用改版都欠一笔同步债。
- 320px 无横向溢出；`prefers-reduced-motion` 下无动画（若加入场动画，一处即可、克制）。
- 不加 JS 交互除非确有必要（悬停即可的不做点击）。
