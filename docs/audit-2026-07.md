## 0. 总评

工程质量高（模块边界清晰、a11y 基线好、16 个 postmortem 换来的 iOS 稳定性），但视觉层是「工程师的默认值」：

- **没有设计令牌层**。`styles.css` 只有 14 个颜色变量 + 2 个尺寸变量（`--control-h`/`--icon-btn`）。字号（11/12/13/14/15/16/17/18/20px）、间距（2–20px 随手取值）、圆角（4/6/8/9/10/12/14/16/20px 九种）、阴影（6 处 ad-hoc rgba 黑）、动效时长（0.12/0.18/0.2/0.25/0.3s）全部散落硬编码。这是「看起来不像顶级产品」的结构性根源：顶级产品的观感 80% 来自节奏一致性，而节奏一致性只能靠令牌。
- **描边驱动的扁平层级**。几乎每个控件都是 `1px var(--border)` 描边 + 微弱底色差，没有海拔（elevation）体系。卡片、输入框、分段控件、mini-btn 视觉权重相同，眼睛没有休息点也没有焦点。
- **关键动效缺失**。表单 sheet 的 `.form-sheet-panel` 声明了 `transition: transform/opacity`，但打开路径是 `hidden` 切换（`display:none → grid`），transition 从不触发——**sheet 是瞬间弹出的，没有任何进出场动画**（`vv-glide` 只在键盘 settle 时生效）。时间轴增删是整段 `innerHTML` 替换，无过渡。视图切换无过渡。这是与原生 app 手感差距的最大单项。
- **信息呈现是「清单」不是「结论」**。尺子 7px 细条 + 13px 等权重百分比文字行，主线/漏损没有视觉优先级，与用户痛点「看数据不够一眼出结论」直接对应。

## 2. 逐屏审计

### 2.1 Header（`index.html:50-71`，`.hdr`）

- **[IA] 主题三连按钮（自动/亮色/暗色）常驻头部黄金位**。设置一次就不再动的偏好占了第一行 1/3 宽度，挤得 390px 以下站名直接隐藏。顶级产品把主题收进设置或做成单按钮循环切换。→ v33/v34 建议：收进 ⚙ 配置 sheet 或改单图标循环。
- **[IA] GitHub 入口重复两处**：`.hdr-icon-link`（站点 icon 链接到 repo）和 `.github-link`（GitHub 图标）指向同一 URL。删一个。
- **[视觉] `?` 和 `⚙` 是裸文本字形**，与 `ui.js iconSvg()` 的 SVG 图标体系（1.5–2px 线宽、统一 viewBox）不一致，渲染粗细随字体走。`iconSvg` 里已有 `help`/`settings` 两个 SVG 定义但 header 没用它们。
- **[a11y] 良好**：aria-label 齐全，44px 目标。

### 2.2 视图切换 + 日期导航（`.view-box`）

- **[视觉] 三块灰底描边控件堆叠**（view-tabs、‹ 周期 ›、回到今天），全部同权重描边卡片，占据首屏 ~140px 高度。周期文字 14px 与 tab 文字 13px 几乎同大，层级靠位置不靠字重。
- **[阻力] 「回到今天」在 <430px 下独占一整行 44px**。它只在离开今天后才有意义，常驻是死空间。→ 建议条件显示（不在今天才出现），或并入 period-label 点击行为。
- **[动效] 切周期/切视图零过渡**，内容瞬间替换，方向感（上一天/下一天）没有空间隐喻。

### 2.3 「+ 记一条」与「切换活动」

- **[IA] 同一动作两个入口**：全宽 accent 大按钮「+ 记一条」和 tl-head 里的 mini「切换活动」都调 `openForm/switchActivity`（`switchActivity` 只是多了跳回今天）。语义差异（记刚才 vs 开下一段）在续记模型里其实是同一件事。→ v34 收敛为一个入口。
- **[视觉] 大按钮无按压位移/涟漪**，只有 `:active{opacity:.8}`。

### 2.4 尺子（`renderRuler`，`.ruler`）

- **[结论力] 这是「一眼出结论」痛点的现场**：7px 色条 + 一行 13px muted 等权重文字（主线 42% 维持 30% 漏损 8% 未记录 20% 待确认 1h 8h30m）。主线净时长——用户最关心的数字——和「未记录 20%」视觉权重完全相同。→ v35：主线/漏损升为大数字（数字 28–34px + 12px 标签），色条降为辅助，百分比行只保留必要项。
- **[视觉] 色条 `gap:1px` 用背景色透缝**，段与段之间是硬切；`transition: flex 0.3s` 在数据变化时会滑动（好，保留）。
- **[a11y] 色条纯颜色编码无文本替代**（文字行补足了，可接受），但色盲下 maintain 绿/leak 橙相对安全，accent 紫与 track 灰在暗色下对比弱。

### 2.5 时间轴（`renderTimeline`，`.entry`）

- **[视觉] 每条记录右侧常驻两个 44×44 图标按钮（编辑/删除）竖排**，一屏 6 条记录就是 12 个描边图标，是时间轴视觉噪音的最大来源。顶级产品的做法：动作藏进点击（进详情/编辑）或滑动手势，列表面只留内容。→ v34 直接操纵范式下，点段即编辑，常驻按钮可撤。
- **[视觉] 段与段是独立圆角卡片 + 8px 间隙**，「时间连续体」的心智模型被切碎；start 时间 13px muted 在卡片左上，段与段的边界（正是点存储模型的「点」）没有视觉实体。→ v34 直接操纵需要边界成为可见、可拖的对象，这个 DOM 形态要重做。
- **[阻力] 补录链路现状**：空隙段有内联「补一下」（好），实体段有「切一刀」（好），但改一条记录的起点时间要：点编辑 → sheet 弹出 → 滚轮找到分钟 → ✓。四步且滚轮定位慢。拖边界把这条链路变成一步。→ v34 主攻。
- **[动效] `[...items].reverse().map().join('')` 整体重绘**，新增/删除/合并无任何过渡，60s tick 靠 signature 比对避免无谓重绘（工程正确，但有数据变化时依然是跳变）。
- **[a11y] gap 条的「补一下」mini-btn 44px 高度不足**（`.mini-btn min-height:26px`），触控目标偏小；`确认 xx-xx` 同理。

### 2.6 表单 sheet（`renderFormSheet` + `sheet_controller.js`）

- **[动效] 打开无进场动画**（见总评）。关闭走 `hidden=true` 同样瞬断。backdrop 无淡入。这在移动端观感上是「网页弹框」和「原生 sheet」的分界线。
- **[视觉] 表单内部全描边扁平**：时间行、textarea、bucket seg、chips、自定义输入五层控件视觉权重相同；12px muted 标签（归类/标签/自定义标签）三行重复解释同一件事，认知负担大于必要。
- **[阻力] 滚轮 picker（`pickers.js mountWheel`）**：scroll-snap + scrollend 模拟，与 iOS 原生轮的惯性/阻尼/触觉反馈有本质差距（这正是用户「手感不如原生」的主要来源之一）；日期列 12px 文字 40px 行高偏局促；对 90 天前的补录要滚很久（窗口 ±90/+7 天线性列表）。→ v34 直接操纵后，绝大多数时间调整走拖拽，滚轮只剩兜底场景，可考虑退役换原生 `<input type="datetime-local">` 或保留极简版。
- **[阻力] 编辑模式的时间字段永远展开滚轮**（`edit-wheel` 常驻），但多数编辑只是改文字/标签——时间控件应折叠为一行触发器（新建模式已经这么做了：`start-time-trigger`，形态不一致）。
- **[视觉] sheet 头部题注是拼接字符串**（`编辑 · 09:30 · #写代码`）12px muted + 15px 单行截断，信息密但无层级。
- **[a11y] 良好**：focus trap、初焦点收进容器、Cmd/Ctrl+Enter 保存、Esc 关闭、内联错误 `scrollIntoView`。P13–P16 的机制都在。

### 2.7 周/月/年汇总（`renderSummaryRows`）

- **[结论力] 每行是「名字 + 总时长 + 色条 + 四个百分比」**，31 行月视图就是 31 条等权重色条——扫一眼找不到「哪天垮了」。缺少按主线占比/时长的视觉编码（如色条高度、数字字重、达标底色）。→ v35 之后的调研项（用户已决定本轮只做今日大数字，这里只记录）。
- **[视觉] `.sum-row:active` 才出描边**，hover 无反馈（桌面）。
- **[阻力] 下钻单向**：月 → 日后，回月视图要点视图 tab（没有面包屑/返回），方向感靠记忆。

### 2.8 Footer（摘要/备份）

- **[IA] 两个低频动作常驻 sticky 底栏**，永久吃掉 52px+safe-area 高度。备份是月级动作，摘要是天级动作，权重却与时间轴同屏常驻。→ v34 结构红线重写时评估：收进 header ⚙ 或保留但降为单行更矮形态。（结构性红线「footer sticky 文档流」允许重写，精神红线不涉及。）

### 2.9 帮助 / 配置 / 备份 / 导入平移 sheet

- 内容合格，视觉同表单 sheet 的扁平问题。配置 sheet 的 `cfg-row`（名称输入 + 免确认 checkbox + 桶下拉）在 390px 下换行后对齐散。
- 帮助文案是产品亮点（模型解释清楚），v33 重排版式即可，不动内容。

### 2.10 横切项

- **[动效] 全局无按压反馈体系**：所有按钮只有 `:active` 换底色/透明度，无 scale/位移，无统一时长曲线。
- **[视觉] 亮色主题是暗色的逐变量翻译**，阴影、描边对比未按亮色调过：亮色下 `--border #d8dae2` 与 `--input #eef0f4` 对比过弱，控件边界发糊；`box-shadow rgba(0,0,0,.22)` 的 tooltip 在亮色下过重。
- **[a11y] 对比度抽查**：暗色 muted `#7878a0` on card `#1a1a28` ≈ 4.5:1 压线；12px muted 大量用于操作性文字（mini-btn、时间），小字+压线对比是可读性风险。v33 色阶要把「12px 以下不用压线对比色」定为规则。
- **[隐私/治理] 无问题**：无 title=、图标语义符合红线、无设备名补丁。

## 4. 保留项（明确不要在重构中丢掉）

- 续记模型的默认值链路（`defaultFormTimestamp` → 占位条 → settle 点），用户确认新建步骤不是痛点。
- P12–P16 全部机制：先几何后揭露、settle/teardown、vv 事件风暴 settle、boot latch 收窄、`.sheet-closing` 护栏。除非结构性替代（如全屏页），不许裸删。
- focus trap / 初焦点 / Cmd+Enter / Esc / 内联错误滚动可见。
- 4 桶配色的语义映射（accent=主线紫 / maintain=绿 / leak=橙 / track=灰），用户已有肌肉记忆，v33 调的是色值品质不是语义。
- tooltip 体系（800ms hover / focus-visible 立即 / 触屏不依赖 hover）。
