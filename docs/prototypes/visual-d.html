为什么不会四不像

「像不像一套」取决于决策规则是否唯一，而不是灵感来自几个产品：
· 一套圆角家族：12 控件 / 16 卡片 / 22 sheet，全站无例外；
· 一套海拔模型：B 的「无描边、暖阴影两级」，C 语法的 cell 分组也改用 inset 底而非 iOS 分隔线体系；
· 一套字阶：13/14/15/18/22/32，取 B 与 A 的中间档补密度；
· 一套控件语法：所有弹层=抓手+取消/完成+cell 组，所有列表=左缘桶色卡。
风险点只有一个：C 的「取消/完成」文字按钮替代现行「✓/撤销图标」， 这属于已获准重写的结构性红线（图标语义红线只约束仍在用图标的地方，不受影响）。

与 B 的密度差异

卡片 padding 13→9px、间隙 9→6px、正文 16→15px、时间/时长/标签并入单行 meta； 正文单行截断，全文进编辑页。iPhone SE 一屏约 7 条（B 约 5 条）。
更短的段不再缩高度——最低一行卡（约 46px），时间失真由左槽胶囊的真实刻度补偿。

摘要/备份的去向

底栏退役，右上角「···」打开「更多」sheet（下方快照）：摘要一击即复制； 备份是子组（复制/下载/导入/分享），沿用 cell 语法。低频动作不再常驻 52px 底栏。
