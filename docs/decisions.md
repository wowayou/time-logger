## D1 · 版本回退 / 快照（暂不做）

**评估日期**：2026-06-30 · **状态**：不落地

**现状真相**：
- 数据只有单一状态 `localStorage['timelog.v1']`，**没有历史/快照**。
- 导入是**按 id 合并**（`src/storage.js` `mergeImportedEntries`），**不是替换**。
- 因此**当前事实上没有「回退」能力**：导入旧备份只能并回旧条目，无法移除「旧备份里没有、但之后新增/删除过」的条目。把「导入旧备份」当回退用会得到错误结果。

**决策**：满 28 天真实记录前不做。回退/快照属于状态历史管理，紧邻被锁死的「可扩展/同步」边界，收益不抵复杂度与跑题风险。

**若将来真要做（不是现在）**：最小真回退原语＝「导入时提供『替换而非合并』选项」（一次性整体替换当前 `entries`），而**不是**引入多版本快照栈或云端历史。先有这个原语，配合现有「下载 JSON」就能手动回退，无需新数据模型。

## D2 · 付费转型 / 多人维护（暂不设计）

**评估日期**：2026-06-30 · **状态**：不落地

**疑问**：之后若转付费、且有其他开发者帮忙维护，该怎么处理？

**回答（付费与维护是两件独立的事）**：
- **付费**：时间尺是本地优先、零后端、无账号的静态 PWA，没有可计费的服务端。任何直接计费都会撞「无后端 / 无账号 / 无云同步」铁律。可设想的最小侵入路径是「赞助 / 一次性买断 + 可选云备份增值」，但这要等 v2 之后、且求职有实质进展再议。
- **多人维护**：靠**许可证 + `CONTRIBUTING` + 治理文档**约束，而不是改运行时结构。许可证已于 v31 落地（见下「许可证：已落地 AGPL-3.0」）；作为唯一著作权人保留商业双许可权，暂不需要 CLA。

**决策**：**许可证部分已落地**（AGPL-3.0，保护性、耐久、便宜）；**付费转型与云备份增值仍 gated**——现阶段（pre-28 天、求职未见实质推进）不设计、不落地，仅记录该决策本身。重申最大风险＝把「付费/治理规划」当成逃避面试推进的镀金。

### 许可证：已落地 AGPL-3.0（2026-07-01，v31）

**选择**：`AGPL-3.0-or-later`（非 MIT），配「作者保留商业双许可」姿态。`LICENSE` 全文 + 每个运行时文件 SPDX 头 + README「许可证与版权」节 + `CONTRIBUTING.md` 均已落地。

**理由**：
- **AGPL §13 堵住 SaaS 闭源克隆**：客户端 JS 本就完全可读，MIT 等宽松许可等于放任他人闭源套壳做成付费服务。AGPL 的网络条款要求「对外提供服务即须开源」，是对纯前端项目为数不多有牙齿的威慑。
- **唯一著作权人保留商业双许可**：现在只有一个作者，无外部贡献，可无 CLA 地在 AGPL 之外另行授权；这为将来「一次性买断 / 可选云备份增值」留了法律口子，又不必现在就设计付费。
- **法律威慑 + git 溯源即最优解**：前端代码无法技术性防拷贝，务实止损＝清晰许可（威慑）＋ 公开 git 历史（作者身份与时间线的可追溯证据）。

**未来可选项（备注，不做）**：软件著作权登记、商标注册——都属「求职有实质进展后再议」，此处仅登记备选，避免日后追溯。

## D3 · 推广策略（gated，暂不主动执行）

**评估日期**：2026-07-01 · **状态**：不落地（推迟主动推广）

**背景**：需求里含「写推广文案 / 选渠道」。主动社区推广正是 `CLAUDE.md`「别镀金」点名的 #1 风险——**用运营/推广动作逃避面试推进**的高发区。经评估重框为「求职证据工作」：本轮只做便宜、耐久、保护性的产出（许可证、博客草稿），**主动推广推迟**到博客产物就位或出现真实使用里程碑之后。

**决策**：现阶段不主动推广。先让「开发博客」作为可写进简历/面试的工程叙事产物落地；等有真实用户信号或求职实质推进后再谈渠道执行。

**渠道建议（就位后再用，不是现在）**：
- **中文社区先拿真实用户**：少数派 / 即刻 / V2EX / 掘金等，受众与「本地优先、隐私、零依赖」调性契合，且应用界面是中文，落地摩擦最小。
- **英文社区用「博客 dev-story」当内容源**：应用 UI 是中文，不适合直接向英文用户推应用；但**工程故事**（零依赖无构建 PWA、红线治理、postmortem 根因文化）可作为英文 dev-story 内容源投递 —— 正是「英文版当内容源、而非推应用」的设想。

**可核实的事实卖点（避免夸大，仅这两条已核对属实）**：
- **不主动保持屏幕常亮**：应用未使用 Wake Lock，不会强制屏幕常亮，符合「轻量、不打扰」定位。
- **完整备份是 Markdown 摘要 + JSON 导出**：摘要导出为 Markdown（可直接贴给 AI 复盘），完整备份为 JSON（复制/下载/分享），均在本地完成、不上传。

**重申**：以上属 gated 规划，登记以备日后，不构成本轮开工项。最大风险＝把「推广规划」当成逃避面试推进的镀金。
