# 使用与理念

## 为什么存在

时间尺是**尺子，不是产品**。

它只有一个用途：让你在求职期间每天看清"时间进了哪里"，从而让你诚实面对"求职推进"到底占了多少。它不是效率应用，不是习惯打卡，不是要展示给别人的东西。

**v2 锁死规则**：在累计 28 天真实记录之前，不做可扩展分类法、不做人类报表、不做图表。数据还没有，功能就是镀金。

**最大风险**：用打磨这个工具来逃避面试推进。每一次"再加个功能"出现在脑子里，先问：目标公司的约面今天推进了吗？

## 怎么用

### 记一条

点「+ 记一条」→ 滚轮选时间（默认当前）→ 写做了什么 → 选标签 → 「记下来」。

**标签含义：**
- `求职推进`：投简历、打电话、准备面试材料、实际面试
- `研究·学工具·逃避`：学工具、刷文章、研究方法论——这里是逃避的藏身处
- `未知`：不记得或不想标，系统自动计为"未记录"
- `杂`：吃饭、睡觉、家务等

### 补录（滚轮）

开会结束后才想起来记？点「+ 记一条」，滚动日期列到今天，滚动时/分到开始时间，记下来。桌面端可用下方输入框精确输入或粘贴 `YYYY-MM-DD HH:mm`。

方向键也管用：聚焦到滚轮列后，↑↓ 每次移动一格。

### 看尺子

顶部彩条 = 今天截止现在的时间分布。紫色=求职推进，绿色=其他，灰色=未记录（含超过 3 小时没记录的空段）。

### 切亮暗色

标题栏右侧分段按钮可在自动（跟系统）、亮色、暗色之间切换。选择跨重启保持。

### 导出备份

底部五个按钮：
- **摘要**：复制当前视图 Markdown，包含生成时间、周期、统计比例和明细，可直接贴给 AI 做复盘；不导出完整历史，不包含内部 id
- **复制**：把 JSON 复制到剪贴板，作为完整文本备份
- **下载**：通过浏览器保存带时间戳的 `.json` 文件；具体位置由浏览器或系统设置决定
- **导入**：从 `.json` 文件恢复，按 id 合并（不会覆盖本机新增的条目）
- **分享**：设备支持 Web Share 时打开系统分享面板，优先分享 JSON 文件；不支持时隐藏按钮，请用「下载」备份 JSON

网页不能强制指定下载路径，也不能保证每次弹出原生位置选择器。建议每周下载一次备份；`localStorage` 被清除或换设备时靠这个恢复。

### GitHub Pages 和多设备

可以只把 `time-logger/` 作为独立仓库发布到 GitHub Pages。不要发布父目录，也不要提交导出的 `timelog-*.json`、真实记录 JSON、截图、`toolkit/`、`archive/` 或具体个人线索。

Pages 只提供应用入口，不提供云同步。其他设备打开同一个 URL 后，本地 `localStorage['timelog.v1']` 仍是独立的；需要迁移时，用旧设备「下载」JSON，再在新设备「导入」。

### 装到主屏

iOS Safari：分享 → 添加到主屏幕。之后图标直接打开，断网也能用。

Android Chrome：菜单 → 添加到主屏幕。应用已包含 PNG、maskable 和 Apple touch 图标；安装入口是否出现仍由浏览器根据访问环境判断。

## 不做什么

- 不跨设备同步（没有服务器）
- 不做复杂报表或更多图表（v2，等 28 天数据）
- 不做提醒推送
- 不引入任何依赖或构建工具
