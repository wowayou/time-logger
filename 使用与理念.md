# 使用与理念

## 为什么存在

时间尺是**尺子，不是产品**。

它只有一个用途：让你在求职期间每天看清"时间进了哪里"，从而让你诚实面对"求职推进"到底占了多少。它不是效率应用，不是习惯打卡，不是要展示给别人的东西。

**v2 锁死规则**：在累计 28 天真实记录之前，不做可扩展分类法、不做人类报表、不做图表。数据还没有，功能就是镀金。

**最大风险**：用打磨这个工具来逃避面试推进。每一次"再加个功能"出现在脑子里，先问：目标公司的约面今天推进了吗？

## 已经有「屏幕使用时间」了，为什么还要它

iPhone 的「屏幕使用时间」和各种自动统计工具，回答的是「**设备**被用了多久」；时间尺回答的是「**你的一天**去了哪里」。差别有四个：

- **覆盖面**：屏幕使用时间只看得见屏幕亮着的部分。面试、打电话、睡觉、吃饭、通勤、纸质阅读、发呆——一天里大量时间不在任何屏幕上。时间尺记录整条 24 小时时间轴，说不清去向的时间被显式标为「未记录」，而不是消失。
- **语义**：自动统计知道你用了浏览器 2 小时，但分不清那是「准备面试」还是「刷资讯逃避」。同一个 App 既能承载主线也能承载偏航；主线/维持/偏航是相对你目标的价值判断，只有你自己能标，自动埋点原理上做不到。
- **记录本身就是干预**：自动统计是事后旁观，看完就完；手动记一条，是在当下面对「刚才这一阵去了哪」。诚实成本前置，这正是尺子的用途。
- **数据主权**：屏幕使用时间的数据在系统和 iCloud 里；时间尺的数据只在本机 `localStorage`，无账号、不上传，导出 JSON 自己拿走。

两者不冲突：屏幕使用时间适合回答「哪个 App 上瘾」，时间尺回答「今天主线推进了多少」。

### 断网可用吗

可用。时间尺是离线优先的 PWA：首次联网打开一次、让 Service Worker 缓存全部运行时文件之后，断网也能正常打开——记录、编辑、补录、统计、摘要、复制、下载备份全部在本机完成，本来就不需要网络。只有两件事需要联网：第一次加载，和获取新版本更新。

## 怎么用

### 记一条

点「+ 记一条」→ 确认这段的开始时间（默认续所看日期的最后记录；空日从 00:00 开始）→ 写做了什么 → 选标签 → 「记下来」。

**标签含义：**
- `求职推进`：投简历、打电话、准备面试材料、实际面试
- `研究·学工具·逃避`：学工具、刷文章、研究方法论——这里是逃避的藏身处
- `未知`：不记得或不想标，系统自动计为"未记录"
- `杂`：吃饭、睡觉、家务等

### 补录（滚轮）

开会结束后才想起来记？点「+ 记一条」，应用会从所看日期的最后记录或空占位条继续；空日从 00:00 开始。需要补到更早的一段时间时，填「起点」和「终点」两个时间——这段会作为新标签切进对应的已有段里，段的其余部分自动补回原来的标签，不牵连其它段。时间轴上每一段都有「补 / 切」按钮，可直接对着那一段补录或切分。桌面端点击日期时间按钮可用 popover 日历和时分步进选择，也可用下方输入框精确输入或粘贴完整日期时间，例如 `YYYY-MM-DD HH:mm`、`2026/6/28 9:5`。只写 `9:30` 不会保存，因为缺少日期。

方向键也管用：聚焦到滚轮列后，↑↓ 每次移动一格。

### 看尺子

顶部彩条 = 今天截止现在的时间分布。紫色=主线，绿色=维持，橙色=偏航，灰色=未记录。统计先按分钟累加，彩条按分钟比例显示；百分比只是展示文案，不参与统计，也不会为了显示而强行凑满 100%。

明确标签但超过 3 小时、且不是 `longOk` 的已关闭段会先显示为"待确认"并计入未记录；默认只有“睡觉”不要求确认。按钮会显示具体起止时间，确认后才按标签统计。确认只绑定这一段的起点和下一条记录时间，补录中间记录或修改相邻时间后会自动失效，需要按新形成的段逐段确认。如果只是把同一时间段改成另一个明确标签，确认仍对这段时间有效。最后一条进行中的超长段不能确认，等下一条记录关闭后再确认。

桌面端把鼠标停在有提示的按钮上，约 800ms 后显示自定义 tooltip，移开立即隐藏；键盘 Tab 聚焦到图标按钮时立即显示。触屏点击不触发 hover tooltip。

### 切亮暗色

标题栏右侧分段按钮可在自动（跟系统）、亮色、暗色之间切换。选择跨重启保持。

### 导出备份

右上角「···」更多菜单里有这些本地动作：

- **摘要**：复制当前视图 Markdown，包含生成时间、周期、统计比例和明细；不导出完整历史，不包含内部 id
- **复制 JSON**：把完整 JSON 复制到剪贴板
- **存储备份**：生成带时间戳的 `.json` 文件；iPhone/iPad 优先打开系统面板，由你明确选择「存储到文件」和目录，其他浏览器走下载
- **导入**：整批预检后按记录安全合并；冲突会全部列出，逐条处理完并复核本机最新状态后才一次性写入
- **分享备份**：优先分享 JSON 文件，再退化为文本分享或下载；按钮始终可见，取消分享不会暗中下载

「完整备份」只表示当前记录和标签配置被完整导出。导入不会删除备份里不存在的本机记录，所以它不是精确恢复、版本回退，也不是“用备份整体替换当前状态”。`localStorage` 仍是唯一数据源；建议定期把 JSON 存到自己能确认的位置。

### GitHub Pages 和多设备

可以只把 `time-logger/` 作为独立仓库发布到 GitHub Pages。不要发布父目录，也不要提交导出的 `timelog-*.json`、真实记录 JSON、真实截图、`toolkit/`、`archive/` 或具体个人线索。

Pages 只提供应用入口，不提供云同步。其他设备打开同一个 URL 后，本地 `localStorage['timelog.v1']` 仍是独立的；需要迁移时，用旧设备「存储备份」JSON，再在新设备「导入」。导入只合并，不清空新设备已有记录。

### 装到主屏

iOS Safari：分享 → 添加到主屏幕。之后图标直接打开，断网也能用。

Android Chrome：菜单 → 添加到主屏幕。应用已包含 PNG、maskable 和 Apple touch 图标；安装入口是否出现仍由浏览器根据访问环境判断。

### 维护检查

改动代码或文档后，先跑：

```bash
python3 scripts/project_audit.py
python3 scripts/confirm_logic_smoke.py
npm run test:ui
git diff --check
```

第一条检查 PWA 版本、缓存、图标、tooltip/icon 和隐私红线；第二条用真实 ES modules 压测超长段确认、4 桶统计和日边界；第三条跑响应式 UI smoke。

## 不做什么

- 不跨设备同步（没有服务器）
- 不做复杂报表或更多图表（v2，等 28 天数据）
- 不做提醒推送
- 不引入任何依赖或构建工具
